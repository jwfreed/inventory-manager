# Documentation Index

This directory is organized by current purpose. Canonical, current docs live in `architecture/`, `inventory/`, and `engineering/`. Historical plans, prompts, and one-off implementation notes have been separated into `decisions/` and `audits/`.

## Canonical

- `architecture/` — structural boundaries and durable design notes
- `inventory/` — domain semantics, workflow contracts, and inventory-specific references
- `engineering/` — contributor guidance, AI guidance, testing docs, and runbooks

## Historical

- `decisions/` — past design decisions, phase plans, remediation plans, and archived planning material
- `audits/` — acceptance records, audit reports, and archived implementation summaries

## Starting Points

- Domain invariants: `inventory/domain-invariants.md`
- Inventory API and workflow references: `inventory/`
- Architecture references: `architecture/`
- Day-to-day developer docs: `engineering/`
- Runbooks: `engineering/runbooks/`

Archived material is not canonical unless a current document explicitly points to it.
