# Skill Selection Matrix

Use this matrix after reading `AGENTS.md` and inspecting `.skills/`. Apply every required skill for the task, then add optional supporting skills when the risk profile expands.

| Task type | Typical examples | Required skills | Optional supporting skills | Primary risk if skipped |
| --- | --- | --- | --- | --- |
| Domain or state semantics change | new inventory state, changing availability meaning, altering transfer semantics, redefining adjustment behavior | `inventory-domain-guardrails`, `inventory-implementation-planner` | `inventory-test-strategy`, `inventory-review-checklist`, `inventory-ui-flow-guardrails` | Silent reinterpretation of stock truth or invalid state transitions |
| Cross-layer implementation work | workflow feature spanning schema, service, API, UI, reporting, or migration boundaries | `inventory-implementation-planner` | `inventory-domain-guardrails`, `inventory-test-strategy`, `inventory-ui-flow-guardrails` | Partial implementation leaves alternate paths inconsistent |
| Review or merge acceptance | PR review, self-review, release review for inventory behavior | `inventory-review-checklist` | `inventory-domain-guardrails`, `inventory-test-strategy`, `inventory-ui-flow-guardrails` | Unsafe inventory logic is approved because the review stayed superficial |
| Test design or test expansion | adding coverage for receipt, transfer, counting, adjustment, concurrency, migration, or action gating | `inventory-test-strategy` | `inventory-domain-guardrails`, `inventory-ui-flow-guardrails`, `inventory-review-checklist` | Happy-path tests miss failure handling, replay, or concurrency corruption |
| Inventory UI workflow change | state badges, action gating, status copy, confirmation modals, partial-completion messaging | `inventory-ui-flow-guardrails` | `inventory-domain-guardrails`, `inventory-implementation-planner`, `inventory-test-strategy` | Operators see misleading states or can trigger invalid actions |
| Migration or historical data reinterpretation | backfill, default changes, field remap, balance derivation change | `inventory-implementation-planner`, `inventory-domain-guardrails` | `inventory-test-strategy`, `inventory-review-checklist` | Historical stock meaning changes without an explicit integrity plan |
| Quantity-affecting workflow addition | receiving, putaway, transfer, pick, ship, consume, quarantine, count, adjust | `inventory-domain-guardrails`, `inventory-implementation-planner`, `inventory-test-strategy` | `inventory-ui-flow-guardrails`, `inventory-review-checklist` | Workflow shortcuts bypass required states and ship without failure-path coverage |
| Audit or discrepancy work | adjustment reasons, override paths, count investigation, unblock flows, audit exposure | `inventory-domain-guardrails`, `inventory-review-checklist` | `inventory-test-strategy`, `inventory-ui-flow-guardrails` | Corrections become unexplainable after the fact |
| Reporting or derived metric change | inventory snapshot field, KPI rollup, replenishment position display, analytics label | `inventory-implementation-planner` | `inventory-domain-guardrails`, `inventory-ui-flow-guardrails`, `inventory-review-checklist` | Derived values get mistaken for system-of-record truth |
